Getting Started Deep Dive: Viewer Experience
Deep Dive

What Viewers can and cannot do

The Viewer role sees a fundamentally different product than Creators — this gap is one of the most under-documented areas in Sigma.

The Viewer experience explained (5 min) Modes, permissions, and what changes at each access level

The three workbook modes

Every Sigma workbook can be accessed in three modes — but not every user gets access to all three. This is the most important concept for understanding the Viewer experience.

ModeWho can accessWhat you can do
View All users (including Viewers) See the published version of the workbook. Interact with controls, filters, and drill-downs that the builder set up. Cannot make any permanent changes.
Explore Explorer license and above Make personal, temporary changes — add columns, sort, apply additional filters. Changes are private and not saved to the published workbook unless explicitly shared as a saved view.
Edit Creator license and above (with edit permission) Permanently modify the workbook structure, add elements, change data sources, publish changes.
Key distinction: Viewers are locked to View mode only. They cannot enter Explore mode. If a Viewer clicks on a table element, they see data — but the editing toolbar doesn't appear and the formula bar is hidden.

Step 1 of 5

Navigate a shared workbook

When a workbook is shared with a Viewer, they access it through a direct link or from the Documents section of their Sigma home screen — wherever the workbook has been shared with them.

Viewers see the published version only. If a Creator has unpublished changes (a draft), Viewers cannot see those changes. The workbook always opens on the first page unless the Viewer has saved a custom view that sets a different default.

The left rail for a Viewer is simplified — they see Home, Documents, and any shared content, but no Administration, no Connections panel, and no "Create new" button.

Screenshot: Workbook in View mode — no edit toolbar

Step 2 of 5

Interact with controls and filters

Controls — listboxes, date pickers, sliders, text inputs — are the primary way Viewers interact with a workbook. They can change filter values, select options, and update what data is displayed, all without entering edit mode.

Controls work immediately: when a Viewer changes a listbox selection, any elements scoped to that control update in real time. The Viewer is not editing the workbook — they're parameterizing a query that runs against the live warehouse.

Builders can choose which elements are affected by each control using the control's Targets setting. A Viewer never sees this configuration — they just see the result.

Scope matters: If a Viewer changes a control and the chart doesn't update, the chart may not be in the control's target list. This is a builder configuration issue, not a Viewer permission issue.

Step 3 of 5

Drill down into data

Builders can configure click-through actions that let Viewers drill into data without editing. When a Viewer clicks a bar in a chart or a row in a table, an action can:

  • Navigate to another page within the same workbook
  • Navigate to a different workbook entirely
  • Apply a filter to other elements on the page
  • Open a URL (e.g. a CRM record or external system)

Viewers can also maximize any element — click the expand icon in the element's top-right corner to see it full-screen. This is useful for charts that are otherwise small on the page. Press Escape to return to the workbook.

Screenshot: Element maximize button (top-right corner)

Step 4 of 5

Save a personal view

Viewers can save their current filter/control state as a saved view (formerly called bookmarks). A saved view captures the filter selections you have active at that moment — it does not modify the published workbook or affect what other users see.

To save a view: open the Document menu (···) → Custom view → Save this custom view → give it a name. You can set it as your default so the workbook always opens with your preferred filters already applied.

Saved views are personal. They live under your account and are not shared with others unless you explicitly export or share the link.

Common point of confusion: Viewers sometimes think they've changed the workbook when they update controls. They haven't — they've just changed their view of it. Saving a custom view makes that personal state permanent for them alone.

Step 5 of 5

Export data as a Viewer

Viewers with the right permissions can export data in several ways:

  • Download to CSV or Excel: Available via the element's action menu (···) → Download. This exports the data currently visible in that element — with your active filters applied.
  • Export to PDF: Available via the Document menu → Export → PDF. Exports the full workbook as rendered on screen.
  • Send now / Schedule: Admins and Creators can set up scheduled exports to email, Slack, or Google Drive. Viewers receive these as recipients, but cannot configure them.

If export options are missing, your admin may have restricted them at the account type level. This is controlled under Administration → Account types → the specific type's capabilities.
